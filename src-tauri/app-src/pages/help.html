What is this Generator?

This program is a simple way to generate all the JSON files you require for things such as mods, resource packs, and data packs!
You give it some inputs, and it automatically generates the JSON files.

The Basics

How to Use the Generator

Simple! Just put inputs into the boxes on the page and watch the generator create the files!

The id format for Minecraft is, for example, 'minecraft:stone_brick_slab'. It doesn't matter how you input your own blocks into the inputs, as the generator will reformat it as needed. This same rule applies to the mod name, as well as the namespace boxes.

Save Locations and File Paths

  • If you are creating a fabric mod, then you will want to select the src/main/resources.
  • If you are creating a forge mod, you will want to select the topmost folder.
  • If you are creating a resource pack, you will want to select the topmost folder.
  • If there are pre-existing folders, those will be used. Otherwise, the folders will be generated for you.

Datapack support is unlikely. However, the crafting recipes could be useful for your datapacks.

Namespace Input

The namespace box (Texture Namespace, Item Namespace) is used inside the file for the items. The 'Mod Name' box is used for the folder structure and some other cases. This value is used if the namespace box is empty.

Block Name Input

The Block Name is the name of the block you wish to create a JSON file for. The input is automatically formatted as needed. Additionally, if the input has the words 'slab', 'stairs', 'wall', 'pillar', or 'mossy', it will be automatically removed. This happens on every page's 'Block Name' box, except for recipes (the inputs are untouched for maximum control).

The Checkboxes

Every page has 5 different checkboxes. Some exceptions include the recipes, and the advancement page. These checkboxes allow you to efficiently create large amounts of JSON files. Just select these boxes, and upon clicking the Generate! button, it will create the JSON files using the inputs you provided.

There is also a template checkbox, to create a template form of that JSON file, for editing yourself.

File Templates

Each page has an option to generate some file templates along with the other files. To use it, just generate the file, and replace all the content that is surrounded in brackets. For more file examples beyond the provided ones, go to here for the rest of the files the generator provides.

The Pages

Block Models

Each radio button near the top tells you what each one allows you to do. The following bit in brackets tells you what blocks are able to use that setting.

Pillars and walls are unlikely to recieve proper support for multiple textures.

Slabs follow a different scheme for texture placement on the block. The "Top texture" is used for the bottom slab's top, "Side Texture" is used for the sides, and "Bottom Textures" is for the bottom. The top slab uses "West Texture" for the top, "South Texture" for the bottom, and "East Texture" for the sides.

Advancements

As of right now, the advancement page can only create recipe advancements. This may be expanded upon later, see issue #41 here.

The 'Trigger Name' box allows you to create a trigger for the advancement. If you wish to unlock recipes, this should be the item that unlocks the other recipes.

Upon clicking generate, you will create a single JSON file with 1 trigger, and 4 given recipes, 'Stairs', 'Slabs', 'Walls' and 'Pillars.

Recipes

Stonecutter Recipes

  • Item Namespace: If you are using modded blocks, change this to your mod name.
  • Ingredient: This is the name of the ingredient.
  • Result: This is the name of the result in the middle panel.
  • Count: This is how many you get from the recipe. Slabs ignore this, and will give 2 for each block.

Furnace Recipes

  • Experience: Most smeltable blocks will give 0.1 experience points per smelt. Cooking food gives 0.35 points. Copper and Iron give 0.7, Gold gives 1, and Ancient Debris gives 2. See this page for further information.
  • Cooking Time: This box determines the length of the smelting process. This value is measured in ticks, and 200 ticks is the default value.

Simple Crafting Table Recipes

The simple recipe page takes the given inputs and places them inside of the JSON files that can be generated via the checkboxes. It allows for a large amount of variated blocks to be generated quickly.

Custom Crafting Table Recipes

For the other page, the table is controlled by clicking on each of the boxes, to change the value on it. The table shifts, simply due to the CSS setup. The input boxes allow you to give values for each of the given letters in the table.

When you do the 2x2 crafting table, the only open boxes are the first 4. The dropdowns still provide all 9 options, but you can only use the first 4. If a box is empty, or if a letter is used but its input box is empty, it will also be ignored.
